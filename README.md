# FestForecast — 지역 축제 방문객 예측

> 교외 팀 프로젝트 (3인) · 담당: EDA · 체감기온 파생변수 설계 · 검증 설계
> 데이터: 문화체육관광부 축제 통계, 기상청 ASOS · 도구: Python (pandas, scikit-learn)

## 문제

같은 축제도 해마다 방문객 수가 달라져 주최 측이 인력·예산 계획을 세우기 어렵다.
"날씨로 방문객을 예측할 수 있을까?"라는 질문에서 출발했다.

## 데이터

- 공공데이터포털 / 문화체육관광부: 지역 축제 방문객 통계
- 기상청 ASOS: 일별 날씨(평균기온·평균습도·평균풍속)
- CSV로 수집 후 **pandas로 결합·전처리**. 축제마다 다른 기간·지역 표기를 표준화했다.
  - 기간: 일 단위 표기는 개최 기간 날씨를 평균, **월 단위 표기(예: `2024-03`)는 해당 월 전체 날씨를 평균**내어 매칭
  - 지역: `진해시` / `창원시 진해구` 등 제각각인 표기를 alias로 통일

## 접근

1. **베이스라인**: 과거 방문 이력 기반 예측
2. "날씨가 설명력을 더 줄까?" — EDA에서 같은 기온 구간인데 방문객이 크게 갈리는 날들을 발견, 그 날들의 습도·풍속이 달랐다
3. "사람은 측정 기온이 아니라 몸으로 느끼는 날씨에 반응한다"는 가설 → 기온·습도·풍속을 조합한 **체감기온 파생변수 설계** (Steadman apparent temperature 근사식)
4. 시계열 누수 방지: 랜덤 분리 대신 **과거 연도 학습 / 최근 연도 검증**

## 결과 (있는 그대로)

가설: 날씨, 특히 체감기온이 방문객 예측을 개선한다.

검증 (2년 이상 개최된 축제 대상, 학습 208행 / 검증 362행):

| 모델 | 검증 R² |
| --- | --- |
| 과거 방문 이력만 | **0.788** |
| + 기온·습도·풍속 | 0.780 |
| + 체감기온 | 0.780 |

- **날씨·체감기온 변수는 예측을 개선하지 못했다** (오히려 미세하게 하락). 방문객 규모는 **과거 이력에 지배적으로 설명**되었고, **초기 가설은 기각**되었다.
- **원인 진단**: 광역 대표 관측소로 날씨를 근사한 점, 상당수가 월 단위 매칭이라는 점이 겹쳐 **날씨 신호가 희석**된 것으로 판단. 또한 방문객 규모 자체가 축제 고유의 인지도·규모(=과거 이력)에 크게 좌우된다.

## 배운 것 · 한계

- 가설이 기각돼도 **그 과정과 원인 규명이 결과**다.
- **검증 성능이 오르지 않는 변수는 추가하지 않는다** — 변수를 늘려 겉보기 성능을 만들기보다 과적합을 경계했다.
- 한계: 광역 관측소 근사·월 단위 매칭으로 날씨 해상도가 낮음 / 선형 모델이라 극단 구간 비선형 미포착
- 다음 과제: 축제 인근 관측소로 매칭 해상도 상향, 트리 계열 모델로 비선형 보완
