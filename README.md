# FestForecast — 지역 축제 방문객 예측

> 원 프로젝트: 2025.06 – 2025.12 (사이드 프로젝트, 3인 팀 · EDA/파생변수 설계/검증 설계 주도)
> 이 저장소는 계정 재생성으로 인한 재구현본입니다.

## 문제
같은 축제도 해마다 방문객이 달라져 주최 측이 인력·예산 계획을 세우기 어렵다.
사전에 수요를 예측할 도구가 없었다.

## 데이터
- 공공데이터포털: 지역 축제 방문객 3년치
- 기상청: 일별 날씨 데이터
- MySQL 적재·조인 후 Python 전처리 (축제별 상이한 기간·지역 표기를 날짜 단위로 표준화)

## 접근
1. 선형 회귀 베이스라인 — R² 0.70에서 정체
2. 변수 추가(기온·강수량·요일)로도 개선 없음 → **모델 교체 대신 EDA 재검토를 판단**
3. 동일 기온에서 방문객이 2배 갈리는 날들의 습도·풍속 차이 포착
4. "사람은 체감 날씨에 반응한다" 가설 → 기온·습도·풍속 조합 **체감기온 파생변수 설계**

## 검증
- 시계열 누수 방지: 과거 연도 학습 / 최근 연도 검증
- 결과: R² 0.70 → 0.85 (검증 데이터 기준)

## 한계와 후속 과제
- 선형 회귀라 폭염·한파 등 극단 구간의 비선형 패턴 미포착 → 트리 계열 모델이 다음 과제
